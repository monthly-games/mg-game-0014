## 2. 코어 루프 (Core Loop)

```
┌─────────────────────────────────────────────────────────┐
│  1. 실험(런) 시작 - 초기 스킬 드래프트                    │
│         ↓                                               │
│  2. 층별 퍼즐 진행 + 스킬/시너지 획득                     │
│         ↓                                               │
│  3. 런 종료 (성공/실패) → 실험 노트 기록                  │
│         ↓                                               │
│  4. 메타 해금/영구 버프 → 재도전 ─────────→ (1로 반복)    │
└─────────────────────────────────────────────────────────┘
```

### 루프 상세 설명

1. **실험 시작**: 초기 스킬 세트 드래프트로 방향 설정
2. **퍼즐 진행**: 층별 퍼즐 클리어하며 스킬/시너지 수집
3. **런 종료**: 성공 또는 실패, 실험 결과 기록
4. **메타 성장**: 영구 해금으로 다음 런 유리하게 시작

## 3. 재미 축 (Fun Pillars)

### 3.1 빌드/시너지 (Build)
- **목표**: 매 런마다 색다른 빌드를 실험하는 재미
- **핵심 경험**: 이 조합 대박이다! 하는 발견의 순간
- **주요 요소**:
  - 스킬 시너지 시스템
  - 테마별 빌드 (화염/얼음/독/번개 등)
  - 드래프트 시스템
  - 시너지 태그

### 3.2 실험/발견 (Experiment)
- **목표**: 다양한 조합을 시도해보는 탐험적 재미
- **핵심 경험**: 이건 처음 보는 조합인데? 하는 호기심
- **주요 요소**:
  - 랜덤 스킬 풀
  - 숨겨진 시너지
  - 실험 노트 시스템
  - 발견 업적

### 3.3 재도전 (Retry)
- **목표**: 실패해도 다시 하고 싶은 중독성
- **핵심 경험**: 다음엔 다른 빌드로 해봐야지
- **주요 요소**:
  - 짧은 런 시간
  - 메타 progression
  - 다양한 시작 조건
  - 도전 모드

## 4. 디자인 레버 (Design Levers)

### 4.1 드래프트 시스템
| 레버 | 설명 |
|------|------|
| 랜덤/드래프트 | 매 판마다 스킬 조합이 완전히 달라지는 랜덤/드래프트 시스템 설계 |
| 구현 방향 | 초기 3스킬 드래프트, 층 클리어 시 3지선다, 숍에서 스킬 교환 |

### 4.2 시너지 시각화
| 레버 | 설명 |
|------|------|
| 명시적 태그 표현 | 시너지 세트를 명시적으로 표현(아이콘/태그)해 전략 선택 지원 |
| 구현 방향 | 스킬 카드에 시너지 태그 표시, 현재 시너지 상태 UI, 추천 조합 힌트 |

### 4.3 실험 노트
| 레버 | 설명 |
|------|------|
| 발견 기록/추천 | "실험 노트" 시스템으로 플레이어가 발견한 조합을 기록/추천 |
| 구현 방향 | 신규 시너지 발견 시 자동 기록, 개인 노트 메모, 커뮤니티 공유 |

## 5. 재미 측정 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 런 당 시간 | 15~20분 | 런 시작 → 종료 |
| 일일 런 횟수 | 4회 이상 | 런 시작 로그 |
| 시너지 발견률 | 주 3개 이상 | 신규 시너지 발견 로그 |
| 빌드 다양성 | 상위 5빌드 50% 이하 | 빌드 사용 통계 |
| D7 리텐션 | 30% 이상 | 7일차 잔존율 |
